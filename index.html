Кейс 03 · 2023 · AI / Рекомендации · тех-лид

Умка — движок ранжирования для поиска фрагмента видео по вопросу пользователя.

  • Задача. По вопросу пользователя найти релевантный фрагмент видео в корпусе на сотни часов.
  • Стадия 1 — поиск кандидатов. Вопрос раскладывается в последовательность векторных представлений; по корпусу считается смысловое расстояние → N ближайших фрагментов.
  • Стадия 2 — доборное ранжирование. Для каждого кандидата строится сжатое описание смысла — «заголовок главы».
  • Два сигнала. Вопрос сопоставляется отдельно: с буквальным текстом и со смысловым описанием. Получаем две меры близости.
  • Итоговая оценка. Взвешенная свёртка двух сигналов → число от 0 до 1. Вес смысла выше веса буквального текста — смысл бьёт букву.
  • В проде. 300 запросов/день у частного заказчика. Собственная реализация — без сторонних фреймворков.
Python Векторные модели Смысловой поиск Переранжирование Redis 2023 · NDA

Та же математика — взвешенная свёртка нескольких сигналов с асимметричным весом — лежит в основе любого движка репутационного ранжирования. Это напрямую применимо к задаче DonateOpen: размер пожертвования × репутация донатора.

Почему оркестрация, а не управление людьми
Управление людьми раздувает фиксированные издержки и замедляет цикл «идея → прод». Агентский пайплайн даёт ту же производительность без найма — и это то, почему один человек сегодня = команда из 5.

Быть тех-лидом над агентскими системами: разработка по спецификациям и ограничениям, декомпозиция до проверяемых инвариантов, делегирование LLM. Это та форма лидерства, которая нужна ведущему инженеру в 2026.
